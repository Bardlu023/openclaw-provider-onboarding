openclaw-provider-onboarding

One-line repo description:
Onboard and verify OpenClaw model providers from API keys, keep only working models, and sync Telegram /models visibility safely.

Short description:
This skill helps OpenClaw integrate third-party model providers without blindly trusting provider catalogs. It can discover candidate models, detect API style, apply layered probing, verify which models actually work in OpenClaw, prune noisy model lists, update agents.defaults.models, and roll back on failure.

Suggested GitHub repo tagline:
